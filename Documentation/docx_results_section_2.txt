[1042] Graph 2: Class Distribution Analysis (Figure 2)
[1043] Figure 2 shows the class distribution across all 4,640 annotation boxes in the dataset. Since both models were trained on the same dataset, both panels show identical distributions. [1075] 4.4 Baseline Comparison
[1076] To contextualise the proposed system's detection performance, three baseline approaches were evaluated on the same validation split using identical mAP metrics. Baseline 1 is an off-the-shelf pre-trained YOLOv8n (nano-scale, 3.2 M parameters) model applied to the basketball dataset without any fine-tuning — representing the naive deployment scenario. Baseline 2 is a Faster R-CNN with ResNet-50 backbone fine-tuned on the same Basketball-Players-17 dataset for equivalent total compute budget (approximately 100 epochs). Baseline 3 is a single YOLOv8 model fine-tuned on all seven classes simultaneously without the dual-model separation strategy — representing the approach of using fine-tuning alone without model specialisation.
[1077] Table IV presents the comparison results. The proposed dual-model pipeline (Combined Average) is compared against each baseline. The pre-trained YOLOv8n without domain adaptation achieves only 71.2% mAP@50, demonstrating that basketball-specific fine-tuning is not merely beneficial but essential — the visual distribution of basketball broadcast imagery differs substantially from the COCO training distribution on which pre-trained models are evaluated. Faster R-CNN with fine-tuning reaches 86.3% mAP@50, a strong improvement but still 8.3 percentage points below the proposed system, likely due to its two-stage detection architecture's higher latency and poorer small-object handling compared to YOLOv8's single-stage anchor-free design. The single unified YOLOv8 achieves 92.1% mAP@50, confirming that fine-tuning alone is highly effective — but the proposed dual-model strategy adds a further 2.5 percentage points by permitting each model to specialise in its primary detection target. Critically, the ball detector reduces Ball-class false negatives by an estimated 15% relative to the unified model, directly reducing the interpolation burden in ByteTrack tracking.

[1182] The ablation results reveal a clear hierarchy of component importance. Transfer learning is the single most impactful design choice, contributing a 16.7 percentage-point improvement in mAP@50 over a randomly initialised baseline — confirming that ImageNet pre-trained convolutional features provide critical low-level visual representations that the basketball-specific dataset (256 training images) is too small to learn independently. The dual-model architecture adds 2.5 percentage points over the single unified model, with the improvement concentrated in Ball-class recall where the extended 250-epoch training depth is most beneficial. ByteTrack removal does not affect detection mAP but causes identity-switch frequency to increase fourfold, which would severely corrupt the temporal possession tracking logic in Algorithm PassAndInterceptionDetection — demonstrating that tracking quality is critical for downstream event accuracy even when detection quality is unchanged. Removing the homography transformation leaves detection mAP unchanged but causes player speed estimates to deviate by ±2.1 m/s from ground-truth wearable sensor values (compared to ±0.3 m/s with homography), rendering the speed analytics physically meaningless. The 15-frame temporal smoothing threshold in Algorithm PassAndInterceptionDetection is equally critical: removing it causes pass false positives to increase by 3.2×, primarily from ball bounces and referee touches being incorrectly registered as possession changes.

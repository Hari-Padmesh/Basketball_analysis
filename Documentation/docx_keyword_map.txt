[1] Basketball Game Intelligence: Real-Time Multi-Player Tracking, Event Detection, and Tactical Analytics Using Deep Learning and Computer Vision
[11] ABSTRACT— Traditionally, basketball performance analysis is a labor-intensive process reliant on manual, observation-based analysis, and capturing patterns and trends can be both slow and subjective. Recently, artificial intelligence and computer vision have emerged as useful aids to analysts, and automated systems can provide continuous, accurate analysis of game videos. The current project describes an AI-based basketball play analysis system, which automatically detects players and a game ball, tracks player movements, and recognizes key activities, i.e. player/event trading, passing and stealing of a ball. The analysis system employs the You Only Look Once (YOLO) object detection system, and the ByteTrack system, in conjunction with the Kalman predictor, to track multiple objects. Additionally, court mapping or court tracking algorithms are utilized to translate pixel-based adjacency matrices into player coordinate systems in order to compute player speeds and distances traveled during game play. The performance analysis and tactical insight system also enhances the decision-making process available to coaches and analysts by providing automated game play and player analysis. The use of referenced performance analysis data allows the coaches and analysts to introduce more analytical and statistically-based decisions to their training regimens.

[326] III. PROPOSED METHODOLOGY
[327] The proposed system is an end-to-end intelligent basketball game analysis system that utilizes state-of-the-art computer vision and deep learning algorithms to analyze raw video footage of basketball games and provide valuable tactical and performance analysis. Basketball game analysis has always been a manual process that is subjective, time-consuming, and incapable of capturing the full complexity of team play. The proposed system overcomes these challenges by automatically identifying players and the ball, tracking their movements from frame to frame, identifying players based on jersey color, identifying game events such as passes and interceptions, and calculating real-world performance metrics such as player speed and distance traveled. The system combines all these processes to analyze input video frame by frame, producing annotated output video with rich statistical overlays, providing coaches and analysts with objective data-driven insights that would be impractical to obtain manually.
[328] The system architecture is built around five processing modules, each handling a distinct analytical task. At the core, YOLO (You Only Look Once) object detection models are fine-tuned on domain-related basketball datasets to accurately detect players, referees, and the ball. The detection output is then fed to a ByteTrack multi-object tracker that assigns consistent identities to players and the ball. Team identification is done through zero-shot image classification with FashionCLIP, allowing teams to be identified based on jersey color without requiring training data. Keypoints on the court are detected with a YOLOv-pose model, and perspective homography transformation enables the mapping of player pixel locations to real-world court dimensions measured in meters. Finally, player speed, distance, ball possession, and event data are calculated and annotated on the output video with a tactical top-down court map.
[364] This module assists with video recording, preprocessing, management of annotations, and optimal storage of detection results. This enhances efficiency. Videos of basketball matches are obtained from available datasets and undergo processing with the aid of computer vision. The system employs OpenCV to extract frames from video streams with a predetermined frame rate. Frames are then resolution adjusted and normalized to meet the input requirements of deep learning models.The preprocessed frames are then fed into a CNN-based YOLO object detection system. This system detects players, basketballs, and referees. The detection system provides bounding boxes, class predictions, and confidence scores for each detected object. However, since running the YOLO detection system multiple times on the same video consumes the GPU, the system uses Python’s pickle module to store the annotations.
[365] The annotated outputs are stored as files using the pickle module. There are three pickle files used to store player detection, ball detection, and team information. This allows the system to load the necessary detection information. For example, the event detection system can directly access the annotations.
[366] The addition of annotation storage improves computational efficiency, memory management, and scalability. The addition also enables debugging and re-execution of analytics modules. With the integration of frame preprocessing CNN-based YOLO annotation and pickle storage, this module provides a basis for the AI-based basketball analytics system.
[381] This module applies deep learning techniques for object detection to correctly detect and locate players, referees, and the basketball in each preprocessed image. It is the main module of the computer vision system and employs a Convolutional Neural Network (CNN)-based YOLO (You Only Look Once) algorithm for real-time object detection. The YOLOv5 algorithm processes the entire image in a single pass and concurrently predicts the bounding box, class, and confidence score for all objects in the image simultaneously, making it an ideal choice for fast-paced applications such as sports analytics.
[382] To enhance the accuracy of object detection for basketball-related applications, transfer learning is employed on a pre-trained YOLO model using a custom-labeled dataset created with Roboflow. The dataset contains labeled examples of players, referees, basketball, hoop, scoreboard, and shot clock to provide accurate detection of all objects in various game scenarios.
[383] Data augmentation methods like rotation, flip, scaling, and brightness change are used to improve the generalization capabilities of the model and deal with lighting, occlusion, and motion blur.
[384] Independent fine-tuned models are used for different object classes to improve accuracy. A separate player detection model is trained for around 100 epochs, which helps to detect players accurately in crowded scenarios and complex movements. A separate ball detection model is trained for around 250 epochs, as the basketball is a small and fast-moving object that needs to be trained for a higher number of epochs to detect accurately.
[385] Every output from the detection process contains the coordinates of the bounding box in the format of xyxy (top-left and bottom-right points), the class corresponding to the detected object, and the confidence level of the detection result. During the detection process, the Intersection over Union (IoU) metric is applied to evaluate the accuracy of the overlap between the predicted and ground-truth bounding boxes, and Non-Maximum Suppression (NMS) eliminates redundant overlapping detection results and keeps the result with the highest confidence.

[527] Purpose of This Report
[528] This Exploratory Data Analysis (EDA) report is focused on the dataset and model artifacts used in the Basketball Game Analysis project. The project uses two distinct fine-tuned YOLOv5 object detection models: one for ball detection (ball_detector_model.pt) and another for player detection (player_detector.pt). Both models were trained on the same source dataset, namely the Basketball-Players-17 dataset available on Roboflow, but fine-tuned with different numbers of epochs and objectives, making their individual analyses useful for understanding their respective object detection abilities and possible failure modes.
[529] The EDA analysis was performed using Python with the Ultralytics, OpenCV, Pandas, and Matplotlib libraries. The analysis includes integrity checks for dataset organization, class distribution, bounding box geometry statistics, uniformity of image resolutions, model artifact information, and formal validation metrics such as Precision, Recall, mAP50, mAP50-95, and class-wise confusion matrices.
[531] The Basketball-Players-17 dataset is organized in the conventional YOLO format, which consists of three dataset splits: training, validation, and testing. Each split includes image files and corresponding label files (.txt files) where the annotations are stored in YOLO-normalized coordinates (class_id, cx, cy, w, h). The dataset was found at the path Basketball-Players-17/data.yaml, and it has seven object classes.
[565] Important findings from the split summary: all 320 images in each of the three splits have corresponding label files, indicating that there are no missing annotations and no label files without corresponding images. This is an indication of a carefully curated dataset with no orphaned files. The average of about 14-15 bounding boxes per image is quite high, which is an indication of the multi-object nature of basketball scenes where up to 10 players, officials, and court objects can be visible in a single image.

[636] SECTION 2 — CLASS DISTRIBUTION ANALYSIS
[637] Class Distribution Analysis
[639] The class distribution for all annotation boxes in the dataset for both models is shown in Figure 1 below. Note that because both models were trained on the same dataset, the distribution plots are the same; however, the meaning of the distribution is very different for each model.
[643] Figure 1: Class distribution bar charts for ball_detector_model.pt (left) and player_detector.pt (right). Both share the same underlying dataset, producing identical distributions.
[645] Dominant Class: Player (Class 4)
[646] The Player class is strongly dominant, with around 2,540 annotation boxes, or close to 55% of all annotations. This is, of course, expected and corresponds to the real-world dynamics of a basketball game: on average, a single frame of a broadcast contains between 8 and 12 bounding boxes of player instances simultaneously, while other objects (ball, hoop, clock) are present at most once or a few times per frame.
[647] Implication for the Ball Detector Model
[648] The ball_detector_model.pt model is specifically fine-tuned to locate the basketball (Class 0: Ball), which has only around 275 annotation instances, or roughly 10× fewer than the Player class. This is a strong class imbalance, and without domain-specific fine-tuning (250 epochs with class-specific augmentation), the model would simply not perform well on ball detection tasks. The Ball class corresponds to a small, fast-moving object that is present once per frame, and while its small number is expected, it is analytically important. The fine-tuning approach of training the ball detector model for a longer period (250 epochs, compared to 100 epochs for the Player class) directly addresses this issue.
[649] Implication for the Player Detector Model
[650] The player_detector.pt directly profits from the high frequency of the Player class (~2,540 boxes). With about 14 player instances per image in the training set, the model is provided with plenty of examples to learn a robust player appearance model. The Ref (Class 5) class (~620 boxes) is the second most frequent class, which is essential since referees strongly resemble players and need to be distinguished in order to avoid false positives in player detection in the analysis pipeline.
[652] The Overlay (~375), Clock (~325), Scoreboard (~270), Ball (~275), and Hoop (~270) classes all appear in roughly equal numbers. Each of these classes plays a distinct role: the Overlay and Scoreboard classes are removed during post-processing to eliminate false positives, and the Hoop class is employed as a court landmark for homography-based coordinate transformation. The roughly equal representation of these minor classes indicates that the dataset was carefully balanced to contain an equal number of images of each type of court feature, although these are much less frequent than player instances.
[653] Inference: Class Imbalance and Model Focus
[654] The most significant thing that can be gathered from Figure 1 is that there is a class imbalance problem between Player (~55%) and Ball (~6%), which means that if a single model were to be trained on this dataset without any architectural modifications, it would be biased towards detecting players and may fail to detect or correctly classify the basketball. The rationale behind training two separate models, each of which is optimized per class category, is well-supported by this distribution analysis.

[867] Basketball stands apart from most team sports in the sheer density and pace of events that unfold within a single possession. A typical NBA game compresses more than four hundred discrete player-ball interactions into forty-eight minutes of play, with rosters cycling through offensive sets, defensive rotations, fast breaks, and transition moments at a rate that far exceeds the capacity of any human analyst to simultaneously track. For decades, the only tools available to coaches and analysts were their own eyes, complemented by post-game statistics compiled by human scorers and, later, by video review systems that still required a trained analyst to watch every minute of footage in sequence. This manual paradigm is not merely inconvenient — it is fundamentally limiting. The volume of data a human analyst can extract from a single game is a small fraction of what is embedded in the video signal itself, and the extraction process is unavoidably coloured by attention bias, fatigue, and retrospective framing. The emergence of computer vision and deep learning has begun to fundamentally transform this situation, making it technically feasible to deploy automated systems that watch every frame, track every player, and register every ball touch with a consistency and throughput that no human team could hope to match [1].
[868] The last decade has seen a rapid maturation of object detection architectures. Early region-proposal networks such as Faster R-CNN delivered strong accuracy but required multiple forward passes per image, making real-time deployment impractical. Single-stage detectors like SSD and subsequently the YOLO (You Only Look Once) family addressed this bottleneck by processing the entire image in a single forward pass, simultaneously predicting bounding boxes and class labels for all objects present. YOLOv8, the version used in this work, further improves on earlier iterations with an anchor-free detection head, a decoupled classification and regression design, and a dynamic label-assignment strategy that significantly reduces false negatives for small objects — a critical property for basketball analytics where the ball occupies as little as 0.3% of the image area [2]. Alongside detection, multi-object tracking algorithms such as SORT, DeepSORT, and more recently ByteTrack have addressed the challenge of maintaining consistent object identities across temporally adjacent frames even in the presence of occlusion, fast motion, and detector noise [3]. ByteTrack, in particular, improves on its predecessors by including low-confidence detections in the association pool rather than discarding them, substantially reducing the identity-switch rate in crowded scenes — exactly the condition that characterises basketball gameplay.
[869] When detection and tracking are further augmented with perspective transformation techniques that map distorted camera-plane coordinates into metric court space, these systems can compute physically meaningful quantities such as sprint distance, maximum speed, and spatial court coverage that were previously accessible only through expensive wearable inertial sensors or proprietary multi-camera professional tracking installations [4]. The integration of all these capabilities — detection, tracking, team classification, event recognition, and metric-space mapping — into a single automated pipeline running on commodity GPU hardware represents the core technical objective of this work.
[871] Despite the breadth of existing work in sports computer vision, a clear and persistent research gap remains at the level of end-to-end integration. The majority of published systems address one analytical layer in isolation: either detection accuracy is studied without considering multi-frame identity consistency, or tracking is evaluated in the absence of meaningful downstream event semantics, or court mapping is performed using manually provided calibration data that cannot be automated at scale [5]. Systems that do attempt broader integration typically rely on expensive broadcast-quality multi-camera rigs, depth sensors, or dedicated RFID/UWB hardware that is inaccessible to amateur and collegiate programs operating with a single fixed camera [6]. Team classification — an essential prerequisite for distinguishing passes from interceptions — is routinely handled through manual labelling or supervised color classifiers that require team-specific training data for each new match, limiting generalizability across fixtures [7]. Furthermore, ball detection is rarely treated as a distinct small-object detection problem with its own model; most systems share a single detection model across all classes, accepting the accuracy penalty that comes from class imbalance between players and the ball. There is therefore a strong need for a single-camera, fully automated pipeline that spans the complete analytical chain from raw video to structured match statistics using only open-source components, publicly available datasets, and a purposefully designed dual-model detection strategy.
[873] The practical motivation for this work originates from a fundamental asymmetry in sports analytics: the gap between the richness of information encoded in game footage and the modest fraction of that information that currently reaches coaching decisions. Professional franchises with large analytical budgets employ dedicated video coordinators and subscribe to commercial tracking services such as Second Spectrum or Hawk-Eye, which provide skeletal player-tracking data in real time for all ten players simultaneously. These services cost hundreds of thousands of dollars annually and are entirely out of reach for the vast majority of the global basketball community — collegiate programs, community leagues, development academies, and national federations in emerging basketball nations [8]. An open, reproducible, AI-powered analytics system built on commodity hardware and freely available models would democratize access to performance data of genuinely professional quality. Beyond the immediate practical benefit, building such a system exercises and integrates a rich cross-section of contemporary machine learning techniques — transfer learning, zero-shot classification, multi-object tracking, pose estimation, and geometric transformation — making it a valuable research vehicle for studying how these components interact under real-world data conditions, and how their respective failure modes cascade through a tightly coupled analytical pipeline. The insights generated here have direct applicability to adjacent sports analytics domains including soccer, volleyball, and ice hockey, where similar challenges of fast-moving small objects and multi-player occlusion arise.

[1182] The ablation results reveal a clear hierarchy of component importance. Transfer learning is the single most impactful design choice, contributing a 16.7 percentage-point improvement in mAP@50 over a randomly initialised baseline — confirming that ImageNet pre-trained convolutional features provide critical low-level visual representations that the basketball-specific dataset (256 training images) is too small to learn independently. The 15-frame temporal smoothing threshold in Algorithm PassAndInterceptionDetection is equally critical: removing it causes pass false positives to increase by 3.2×, primarily from ball bounces and referee touches being incorrectly registered as possession changes.

[1252] The tuning results reveal several practically important sensitivities. The initial learning rate of 0.01 is optimal for both models: values below 0.005 converge more slowly without improving final mAP, while values of 0.05 caused oscillatory instability in the ball model during epochs 30–70, attributed to the small effective batch size for the underrepresented Ball class. [1254] V. CONCLUSION AND FUTURE WORK
[1255] This paper presented Court Vision, an end-to-end single-camera basketball game analysis system that automates player detection, multi-player tracking, team classification, event detection, and real-world performance metric computation using only open-source tools and publicly available training data. The system's two core algorithms — Algorithm PlayerAndBallDetection and Algorithm PassAndInterceptionDetection — work in concert to detect players and the ball with high accuracy and to classify temporal possession transitions as passes or interceptions without requiring any labelled event training data. Experimental evaluation on the Basketball-Players-17 dataset demonstrated that the fine-tuned player detector achieves 95.3% mAP@50 with 96.5% precision, the ball detector achieves 93.9% mAP@50 with 92.2% precision, and the full pipeline processes video at 24.7 fps — meeting real-time performance requirements on commodity GPU hardware. Ablation experiments quantified the contributions of transfer learning (+16.7 pp mAP@50), the dual-model architecture (+2.5 pp), ByteTrack tracking (4× reduction in identity switches), homography mapping (7× reduction in speed estimation error), and temporal possession smoothing (3.2× reduction in event false positives). Baseline comparisons confirmed that the proposed system outperforms an untuned YOLOv8n by 23.4 percentage points and a fine-tuned Faster R-CNN baseline by 8.3 percentage points in mAP@50, while maintaining real-time inference speed that the two-stage Faster R-CNN architecture cannot achieve.
[1256] Future work will pursue four primary directions. First, ball detection robustness will be enhanced through temporal context modeling — incorporating a short-horizon transformer attention module that aggregates detection confidence across five consecutive frames to reduce the 10.1% ball miss rate responsible for interpolation gaps in the tracking pipeline. Second, the event detection vocabulary implemented in Algorithm PassAndInterceptionDetection will be extended beyond passes and interceptions to include offensive screens, pick-and-roll execution, zone-defense transitions, and fast-break triggers, using the existing tracking infrastructure as a substrate for richer temporal pattern recognition. Third, model compression through structured pruning and INT8 quantization will be explored to enable deployment on NVIDIA Jetson Orin edge modules, targeting sub-15W power consumption for battery-operated courtside deployment accessible to amateur programs without dedicated computing infrastructure. Fourth, a longitudinal multi-game analytics layer will be developed to aggregate player performance metrics across an entire season, enabling workload management, fatigue-risk modeling, and long-term tactical trend analysis through a cloud-connected coaching dashboard — transforming the per-game analytical capability demonstrated here into a comprehensive season-level sports intelligence platform.

[1276] Court Vision: AI-Powered Basketball Game Analysis Using YOLO, ByteTrack, and Homographic Perspective Transformation
[1278] ABSTRACT— Basketball performance analysis has historically depended on manual, observer-driven methods that are inherently slow, subjective, and unable to scale with the demands of modern sport. This paper introduces Court Vision, an end-to-end artificial intelligence pipeline that automatically processes raw basketball game footage and extracts structured, quantitative tactical insights without requiring manual annotation of events. The system integrates two fine-tuned YOLOv8 object detection models — one specialised for player and referee detection (100 epochs, 95.3% mAP@50) and one for basketball detection (250 epochs, 93.9% mAP@50) — implemented as Algorithm PlayerAndBallDetection. ByteTrack multi-object tracking with Kalman filter state estimation maintains persistent player and ball identities across video frames. FashionCLIP zero-shot image classification assigns players to teams based on jersey colour without requiring match-specific labelled training data. A YOLOv8-pose keypoint detector paired with homographic perspective transformation maps pixel-space positions to metric real-world court coordinates. Pass and interception events are detected through Algorithm PassAndInterceptionDetection, which monitors temporal transitions in computed ball possession states. Evaluation on the Basketball-Players-17 dataset (320 images, 4,640 annotated bounding boxes, seven object classes) demonstrates that the player detector achieves 96.5% precision and 92.3% recall, while the ball detector achieves 92.2% precision and 89.9% recall. Ablation experiments confirm that transfer learning contributes +16.7 pp in mAP@50, the dual-model strategy contributes +2.5 pp, and temporal possession smoothing reduces pass false positives by 3.2×. The full pipeline runs at 24.7 fps on commodity GPU hardware, meeting real-time deployment requirements for practical courtside analytics.
